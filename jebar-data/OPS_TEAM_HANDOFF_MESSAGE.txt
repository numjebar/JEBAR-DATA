JEBAR DATA -> OPS handoff

สรุปสำหรับทีมอีกแอป:

1) ไม่ต้องรอไฟล์ JSON manual แล้ว
ให้อ่านตรงจาก Supabase ของ JEBAR DATA เลย

2) จุดที่ต้องอ่าน:
- table: public.jebar_app_state
- filter: shop_code = jebar
- field หลัก: db

3) master data ที่ให้อีกแอปใช้:
- db.menus
- db.ingredients
- db.packages
- db.recipeBase
- db.batchRecipes
- db.batchRecipeLines
- db.recipeBatch
- db.mediaAssets

4) key กลางที่ต้องใช้ตรงกัน:
- db.ingredients[].id -> OPS ingredients.code
- db.menus[].id -> OPS recipes.code

5) วัสดุสิ้นเปลืองต้องอยู่ใน JEBAR DATA ด้วย
เช่น:
- น้ำยาล้างจาน
- ทิชชู่
- น้ำยาถูพื้น
- ถุงขยะ
- ถุงมือ
- ฝาแก้ว
- หลอด

โดยให้เก็บเป็น master ใน JEBAR DATA ก่อน
แนะนำใช้ code เช่น SUP001, SUP002, SUP003

6) อีกแอปเขียนกลับ DATA ได้เฉพาะ stock zone:
- db.stockItems
- db.stockLots
- db.stockBalances
- db.stockMovements
- db.purchaseEvents
- db.productionEvents
- db.wasteEvents
- db.integrationInbox
- db.integrationOutbox

7) ห้ามเขียนทับ master zone:
- db.menus
- db.ingredients
- db.packages
- db.recipeBase
- db.batchRecipes
- db.batchRecipeLines
- db.recipeBatch
- db.mediaAssets

8) เอกสารอ้างอิง:
- JEBAR_DATA_OPS_ALIGNMENT.md
- JEBAR_DIRECT_SUPABASE_API.md

สรุปสุดท้าย:
JEBAR DATA = master data owner
JEBAR OPS = stock / purchase / production / waste transaction owner
